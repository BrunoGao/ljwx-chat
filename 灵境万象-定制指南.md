## 🚀 使用指南

### 启动 Ollama

1. **安装 Ollama**（如果还没安装）

   ```bash
   # macOS
   brew install ollama
   
   # 或访问 https://ollama.com 下载安装
   ```

2. **启动 Ollama 服务**

   ```bash
   ollama serve
   ```

3. **下载模型**（示例）

   ```bash
   # 下载 Qwen（通义千问）
   ollama pull qwen2.5:7b
   
   # 或其他中文模型
   ollama pull llama3.2
   ollama pull mistral
   ```

4. **查看已安装模型**
   ```bash
   ollama list
   ```

---

### 启动灵境万象

#### 方式 1: 开发模式（推荐用于测试）

```bash
cd ~/work/codes/AI/lobe-chat

# 确保依赖已安装
pnpm install

# 启动开发服务器
bun run dev

# 访问 http://localhost:3010
```

#### 方式 2: 生产构建

```bash
cd ~/work/codes/AI/lobe-chat

# 构建
bun run build

# 启动
bun run start

# 访问 http://localhost:3210
```

#### 方式 3: Docker 部署

```bash
# 构建自定义镜像
docker build -t lingjingwanxiang .

# 运行（连接到本地 Ollama）
docker run -d \
  -p 3210:3210 \
  -e OLLAMA_PROXY_URL=http://host.docker.internal:11434 \
  -e ENABLED_OLLAMA=1 \
  --name lingjingwanxiang \
  lingjingwanxiang
```

## 🧪 功能测试

### 测试 1: Ollama 模型连接

1. 打开灵境万象界面
2. 进入 **设置 → 语言模型 → Ollama**
3. 确认 Ollama 端点：`http://127.0.0.1:11434`
4. 点击 **获取模型列表**，应该能看到已安装的模型
5. 创建新对话，选择 Ollama 模型
6. 发送消息测试

### 测试 2: 知识库功能

#### 准备工作（可选 - 完整知识库需要数据库）

**简化版（使用文件直接注入）：**

1. **上传文件**
   - 设置 → 知识库 → 文件
   - 上传一个 TXT 或 PDF 文件

2. **在对话中启用文件**
   - 创建新对话
   - Agent 设置 → 文件
   - 添加并✅ 启用文件

3. **提问测试**
   - 询问文件相关内容
   - AI 应该基于文件内容回答

**完整版（需要 PostgreSQL + PGVector）：**

```bash
# 1. 安装 PostgreSQL
brew install postgresql@14

# 2. 启动 PostgreSQL
brew services start postgresql@14

# 3. 创建数据库
createdb lingjingwanxiang

# 4. 在数据库中安装 pgvector 扩展
psql lingjingwanxiang -c "CREATE EXTENSION vector;"

# 5. 更新 .env
DATABASE_URL=postgresql://username@localhost:5432/lingjingwanxiang
```

然后可以使用完整的知识库功能：

- 创建知识库
- 添加多个文件
- 自动生成向量嵌入
- AI 主动搜索知识库

## ⚙️ 配置说明

### .env 文件配置

```bash
# ========== Ollama 配置 ==========
# 本地 Ollama 服务地址
OLLAMA_PROXY_URL=http://127.0.0.1:11434

# 启用 Ollama
ENABLED_OLLAMA=1

# 可选：指定自定义模型列表
# OLLAMA_MODEL_LIST=qwen2.5:7b,llama3.2,mistral

# ========== 数据库配置（知识库功能） ==========
# PostgreSQL with PGVector
# DATABASE_URL=postgresql://user:pass@localhost:5432/dbname

# ========== 嵌入模型配置（知识库向量搜索） ==========
# 可以使用 Ollama 的嵌入模型
# OPENAI_API_KEY=ollama  # 特殊标记，使用 Ollama
# 或使用 OpenAI
# OPENAI_API_KEY=sk-your-key

# ========== 开发设置 ==========
NODE_ENV=development
```

## 🔧 问题排查

### Ollama 连接失败

1. 确认 Ollama 服务运行：

   ```bash
   ps aux | grep ollama
   ```

2. 测试 Ollama API：

   ```bash
   curl http://127.0.0.1:11434/api/tags
   ```

3. 检查 `.env` 配置是否正确

### 知识库功能不可用

1. **简化版**：只需上传文件并在 Agent 中启用即可
2. **完整版**：需要配置数据库 + 嵌入模型

### 依赖安装问题

```bash
# 清理并重新安装
rm -rf node_modules
pnpm install
```
