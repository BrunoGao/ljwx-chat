## 🎯 最新修复方案

### 核心改进

创建两个独立的 S3Client：

```typescript
class S3 {
  private client: S3Client; // 使用 S3_ENDPOINT - 内部操作
  private presignClient: S3Client; // 使用 S3_PUBLIC_DOMAIN - 生成预签名 URL
}
```

**优势**:

- ✅ 内部操作使用容器可访问的地址
- ✅ 预签名 URL 使用浏览器可访问的地址
- ✅ 签名基于正确的 endpoint，不会失效

## 🎓 技术原理

### 为什么需要两个客户端？

| 场景            | 使用的客户端    | Endpoint                    | 访问者   |
| --------------- | --------------- | --------------------------- | -------- |
| 上传 / 删除文件 | `client`        | `host.docker.internal:9002` | 容器内部 |
| 生成预签名 URL  | `presignClient` | `192.168.1.83:9002`         | 浏览器   |

### 错误方案 vs 正确方案

**❌ 错误方案**（之前的修复）:

```typescript
// 生成 URL 后替换域名 → 破坏签名
const url = await getSignedUrl(client, command);
return url.replace('host.docker.internal', '192.168.1.83');
```

**✅ 正确方案**（当前修复）:

```typescript
// 使用正确的 endpoint 生成 URL → 签名正确
return getSignedUrl(presignClient, command);
```

### AWS 签名机制

AWS Signature V4 包括：

- HTTP 方法
- **完整 URL（包括 host）** ← 关键
- Headers
- Payload

修改 URL 会导致签名失效 → 403 Forbidden
