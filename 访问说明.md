## 💡 解决方案

### 方案 1: 使用客户端模式（推荐 - 快速测试）

**优点:**

- ✅ 无需认证，立即可用
- ✅ Ollama 模型正常工作
- ✅ 界面完整功能

**缺点:**

- ❌ 数据存储在浏览器中（IndexedDB）
- ❌ 无法使用服务器端知识库（PostgreSQL + PGVector）
- ❌ 数据不能跨设备同步

**配置步骤:**

1. **修改 .env 文件:**

```bash
cd ~/work/codes/AI/lobe-chat

# 编辑 .env，注释掉数据库相关配置
# 将以下行注释掉：
# DATABASE_URL=...
# DATABASE_DRIVER=...
# KEY_VAULTS_SECRET=...
```

或者直接运行：

```bash
cat > ~/work/codes/AI/lobe-chat/.env << 'EOF'
# ============ Ollama Configuration ============
OLLAMA_PROXY_URL=http://127.0.0.1:11434
ENABLED_OLLAMA=1

# ============ Development Settings ============
NODE_ENV=development
EOF
```

2. **重启服务器:**

```bash
# 终止当前服务器 (Ctrl+C)
# 或者
lsof -ti:3010 | xargs kill -9

# 重新启动
cd ~/work/codes/AI/lobe-chat
bun run dev
```

3. **访问应用:**

打开 <http://localhost:3010>

### 方案 2: 配置认证（完整功能）

**优点:**

- ✅ 完整的服务器端知识库功能
- ✅ PostgreSQL + PGVector 向量搜索
- ✅ 数据持久化和跨设备同步

**缺点:**

- ❌ 需要配置认证系统
- ❌ 设置稍微复杂

**配置步骤:**

#### 选项 A: 使用 NextAuth (Email/Password)

1. **安装 next-auth adapter:**

```bash
cd ~/work/codes/AI/lobe-chat
pnpm add @auth/drizzle-adapter
```

2. **配置 NextAuth:**

在 `.env` 中添加：

```bash
# NextAuth 配置
NEXTAUTH_SECRET=iH3sVoHNBBVacCwVijDWk3MSx5E5VZvGua0xy3XGJ2g=
NEXT_AUTH_SECRET=iH3sVoHNBBVacCwVijDWk3MSx5E5VZvGua0xy3XGJ2g=
NEXTAUTH_URL=http://localhost:3010

# 启用 NextAuth
NEXT_PUBLIC_ENABLE_NEXT_AUTH=1

# 添加一个简单的密码认证
# (这需要额外的代码配置)
```

3. **创建初始用户:**

```bash
psql -d lingjingwanxiang << 'SQL'
INSERT INTO users (id, username, email, is_onboarded)
VALUES ('admin', 'admin', 'admin@local.com', true)
ON CONFLICT (id) DO UPDATE
SET username = EXCLUDED.username;
SQL
```

#### 选项 B: 禁用 tRPC 认证中间件（仅开发环境）

**警告:** 这会完全禁用认证，仅适合本地开发！

1. **临时修改认证中间件:**

编辑 `src/libs/trpc/middleware/userAuth.ts`

```typescript
export const userAuth = trpc.middleware(async (opts) => {
  const { ctx } = opts;

  // 开发模式下跳过认证
  if (process.env.NODE_ENV === 'development') {
    return opts.next({
      ctx: { userId: 'default-user' },
    });
  }

  // ... 其余代码保持不变
});
```

2. **重启服务器:**

```bash
cd ~/work/codes/AI/lobe-chat
bun run dev
```
