# 文件上传功能修复说明

## 问题诊断

您遇到的 "文件上传失败" 错误是因为 **MinIO 服务未运行**。从您的配置可以看到，系统配置了 MinIO 作为文件存储服务，但 MinIO 容器没有启动。

## 快速修复 (推荐)

### 一键启动所有服务

```bash
./start-all.sh
```

这个脚本会自动:

1. 启动 MinIO 服务
2. 创建 bucket 并配置 CORS
3. 启动 LobeChat 应用

### 停止所有服务

```bash
./stop-all.sh
```

## 手动修复步骤

如果自动脚本失败，可以按照以下步骤手动操作:

### 步骤 1: 启动 MinIO

```bash
docker-compose -f docker-compose.minio.yml up -d
```

### 步骤 2: 配置 MinIO

```bash
./setup-minio.sh
```

或者手动配置:

1. 访问 MinIO 控制台: <http://localhost:9003>
2. 登录账号: `minioadmin` / `minioadmin`
3. 创建 bucket `lingjingwanxiang`
4. 设置 bucket 为公开访问
5. 配置 CORS 规则 (允许所有来源)

### 步骤 3: 重启应用

```bash
docker-compose -f docker-compose.lingjingwanxiang.yml restart
```

## 验证修复

1. 访问 LobeChat: <http://192.168.1.83:3210>
2. 尝试上传一个文件
3. 如果成功，问题已解决

**关键配置点:**

- LobeChat 容器通过 `host.docker.internal:9002` 访问 MinIO
- 浏览器通过 `192.168.1.83:9002` 访问 MinIO (用于下载上传的文件)
- 必须配置 CORS 允许跨域访问

## 常见问题

### Q: 为什么需要 MinIO?

A: LobeChat 使用服务端数据库模式时，文件 (图片、文档等) 需要存储在对象存储服务中。MinIO 是一个兼容 S3 API 的开源对象存储解决方案。

### Q: 可以使用其他 S3 服务吗？

A: 可以。您可以使用任何兼容 S3 的服务，如:

- Cloudflare R2
- 阿里云 OSS
- 腾讯云 COS
- AWS S3

只需修改 `docker-compose.lingjingwanxiang.yml` 中的 S3 环境变量即可。

### Q: 生产环境建议？

A: 生产环境建议:

1. 使用专业的 S3 服务 (更稳定，有 CDN)
2. 配置 HTTPS (避免安全警告)
3. 使用强密码
4. 定期备份 MinIO 数据

### Q: 如何检查服务状态？

A:

```bash
# 检查所有容器
docker ps

# 应该看到:
# - lingjingwanxiang (LobeChat)
# - lingjingwanxiang-minio (MinIO)

# 查看日志
docker logs -f lingjingwanxiang
docker logs -f lingjingwanxiang-minio
```
