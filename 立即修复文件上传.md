## 💡 技术解释 (可选阅读)

### 为什么会失败？

Docker 容器内部使用 `host.docker.internal` 访问 host 上的 MinIO, 但浏览器在 host 上，无法解析这个特殊域名。

### 修复做了什么？

在生成预签名 URL 后，自动将内部地址替换为外部地址:

- 内部: `http://host.docker.internal:9002`
- 外部: `http://192.168.1.83:9002`

### 为什么不直接用外部地址？

因为容器内部必须通过 `host.docker.internal` 才能访问 host 服务，这是 Docker 的网络机制。
