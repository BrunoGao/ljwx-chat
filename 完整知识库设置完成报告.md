## 🔍 故障排查

### 问题 1: 依赖安装失败

**症状:** pnpm install 报错或卡住

**解决方案:**

```bash
cd ~/work/codes/AI/lobe-chat

# 清理并重新安装
rm -rf node_modules
rm pnpm-lock.yaml
pnpm install
```

### 问题 2: 数据库迁移失败

**症状:** `bun run db:migrate` 报错

**检查步骤:**

1. 验证数据库连接

   ```bash
   psql -d lingjingwanxiang -c "SELECT 1;"
   ```

2. 检查 .env 配置

   ```bash
   cat .env | grep DATABASE_URL
   ```

3. 手动运行迁移脚本
   ```bash
   cd ~/work/codes/AI/lobe-chat
   MIGRATION_DB=1 node_modules/.bin/tsx ./scripts/migrateServerDB/index.ts
   ```

### 问题 3: Vector 扩展不可用

**症状:** 创建知识库时报错 "vector extension not found"

**解决方案:**

```bash
# 重新启用扩展
psql -d lingjingwanxiang -c "DROP EXTENSION IF EXISTS vector;"
psql -d lingjingwanxiang -c "CREATE EXTENSION vector;"

# 验证
psql -d lingjingwanxiang -c "SELECT extname, extversion FROM pg_extension WHERE extname = 'vector';"
```

### 问题 4: Ollama 连接失败

**症状:** 无法连接到 Ollama 或找不到模型

**解决方案:**

```bash
# 1. 检查 Ollama 服务
ps aux | grep ollama

# 2. 启动 Ollama (如果未运行)
ollama serve &

# 3. 验证模型
ollama list | grep qwen3

# 4. 测试 API
curl http://127.0.0.1:11434/api/tags
```

### 问题 5: 向量嵌入生成失败

**症状:** 文件上传后无法生成向量

**原因:** 需要配置嵌入模型

**解决方案 1: 使用 Ollama 生成嵌入**

```bash
# 下载嵌入模型
ollama pull nomic-embed-text

# 在 .env 中配置
echo "EMBEDDING_MODEL=ollama" >> .env
echo "OLLAMA_EMBEDDING_MODEL=nomic-embed-text" >> .env
```

**解决方案 2: 使用 OpenAI API**

```bash
# 在 .env 中添加
echo "OPENAI_API_KEY=sk-your-key-here" >> .env
```
